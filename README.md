# 🌤️ Weather & Tourism API - Tech Assessment

This project is a backend developed in **Python (FastAPI)** focused on data persistence (CRUD), external API integrations, and data export, fulfilling all the requirements of the Technical Assessment.

## 🚀 Features

The system goes far beyond a simple weather API, offering the following features:

- **Complete CRUD:** Allows creating, reading, updating, and deleting weather records.
- **Data Persistence:** Uses a NoSQL database (**MongoDB Atlas**) for asynchronous storage.
- **Robust Validations:** Strict validation of date ranges and real existence of the searched location through geocoding.
- **External Integrations (Requirement 2.2):**
  - **Open-Meteo:** Provides exact coordinates and daily maximum temperatures.
  - **Google Maps:** Generates URLs pointing to the searched latitude/longitude.
  - **Wikipedia API:** Extracts a clean and encyclopedic summary about the searched city.
  - **YouTube API:** Provides links to tourism videos (with a dynamic fallback if the API Key is not provided).
- **Data Export (Requirement 2.3):** Dynamic routes to download the database in `JSON`, `CSV`, and `Markdown` formats.

## 🛠️ Technologies Used

- **Web Framework:** [FastAPI](https://fastapi.tiangolo.com/)
- **Language:** Python 3.13+
- **Database:** MongoDB (via Asynchronous driver)
- **Data Validation:** Pydantic V2
- **HTTP Requests:** httpx
- **Package Manager:** `uv`

## ⚙️ Environment Variables (.env)

Create a `.env` file in the root of the project and add the following variables:

```env
# Connection URL to your MongoDB Atlas
MONGO_URL="mongodb+srv://<username>:<password>@cluster.mongodb.net/?retryWrites=true&w=majority"

# (Optional) YouTube API Key. If left blank, the system will fallback to generic search URLs.
YOUTUBE_API_KEY=""
```

## 💻 Installation and Execution

This project uses [uv](https://github.com/astral-sh/uv) as a package manager for superior performance.

1.  **Clone the repository:**

    ```bash
    git clone <your-repository-url>
    cd pma-technical-assessment-backend
    ```

2.  **Create and activate the virtual environment:**

    ```bash
    uv venv
    source .venv/bin/activate  # Linux/macOS
    # or on Windows: .venv\Scripts\activate
    ```

3.  **Install the dependencies:**

    ```bash
    uv pip install -r requirements.txt
    ```

4.  **Start the development server:**

    ```bash
    uv run uvicorn app.main:app --reload
    ```

The server will be available at: `http://localhost:8000`

## 📖 API Documentation (Swagger)

The interactive and detailed documentation (Swagger UI) is automatically generated by FastAPI. After starting the server, go to:

👉 **[http://localhost:8000/docs](http://localhost:8000/docs)**

From Swagger, you can test all the endpoints (POST, GET, PUT, DELETE) and download the export files directly from your browser.
